# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them through [GitHub Security Advisories](https://github.com/paco/stripe-payout-reconciler/security/advisories/new).

You should receive a response within 48 hours. If the issue is confirmed, a patch will be released as soon as possible, typically within a few days depending on severity.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Any suggested fix (optional)

## Scope

This project handles Stripe API keys. Security issues related to credential handling, output sanitization, or dependency vulnerabilities are taken seriously.

## Disclosure

We follow responsible disclosure. After a fix is released, we will publish a security advisory with credit to the reporter (unless anonymity is requested).
